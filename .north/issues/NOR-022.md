**Context**: Users cannot scroll to the bottom of the issue list in the kanban pane when there are more issues than fit in the viewport.

**Requirements**:
- User should be able to scroll through all issues regardless of list length
- Scrolling should reach the very last issue in the list
- No artificial limits on scroll offset
- Smooth scrolling behavior

**Acceptance Criteria**:
- User can scroll to see all 29 rows when viewport shows 18 rows (5-22)
- Bottom issue is fully visible when scrolled to end
- Scroll offset properly calculated based on total issues and viewport height

**Technical Details**:
- Files: `src/components/kanban-pane.ts` (scroll logic)
- Current behavior: Stops at rows 5-22 out of 29 (can't reach rows 23-29)
- Likely issue: `scrollOffset` max calculation is off
- Check scroll clamping logic in `scrollBy()` method
- Verify viewport height calculation
- May need to account for header/status rows in scroll calculations

**Reproduction**:
1. Create 29+ issues
2. Open TUI
3. Try to scroll down with mouse wheel or arrow keys
4. Observe: stops at row 22, cannot reach rows 23-29
