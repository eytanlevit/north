**Context**: Enable users to install north globally and run it from any directory within a git repository, automatically detecting the project root.

**Requirements**:
- Support global installation via npm (`npm install -g north`)
- Support npx execution (`npx north`)
- Automatically detect project root by finding `.pm` folder at git repository root
- If not in a git repo or no `.pm` found at root, show error message: "Not in a north project. Run 'north init' to create one."
- Work seamlessly with existing local installations
- Ensure sessions/logs save to correct project `.pm` folder

**Acceptance Criteria**:
- User can install globally and run `north` from any subdirectory of a git repository
- Command finds `.pm` folder at git root
- If outside a project or `.pm` not at git root, shows clear error message
- Sessions and logs save to the correct project `.pm` folder

**Technical Details**:
- Files: `package.json` (bin config), `src/project-root.ts` (detection logic)
- Detection strategy:
  1. Find git root using `git rev-parse --show-toplevel`
  2. Check if `{git_root}/.pm` exists
  3. If yes, use that as project root
  4. If no, show error and exit
- Only look at git root (don't walk directory tree for nested .pm folders)
- No global config file for now (keep everything project-specific)
