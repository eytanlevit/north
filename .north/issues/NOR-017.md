The CLI `create` command always generates issue IDs with the `ISS` prefix instead of reading the configured prefix from `.pm/config.yaml`.

## Steps to Reproduce
1. Have a project with `prefix: PMT` in `.pm/config.yaml`
2. Run: `npx tsx src/cli.ts create "Some issue" --priority high`
3. Issue is created as `ISS-001` instead of `NOR-016`

## Root Cause
`cmdCreate()` in `src/cli.ts:169` calls `nextId(cwd)` without passing the prefix. The `nextId()` function in `src/issues.ts:109` defaults to `"ISS"` when no prefix is provided.

The `normalizeId()` function (cli.ts:66-67) correctly reads the config prefix, but `cmdCreate()` does not.

## Fix
Load config in `cmdCreate()` and pass `config.prefix` to `nextId(cwd, config.prefix)`. Same fix likely needed for any other CLI commands that generate IDs.
