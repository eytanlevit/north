## Problem
The background color for user messages in the agent pane is being applied to the entire terminal row, causing it to bleed into the kanban board pane on the right side. This creates visual pollution and makes the kanban pane harder to read when user messages are visible.

## Expected Behavior
- User message background color should only apply to the content within the agent pane boundaries
- The kanban pane should maintain its own independent background styling
- The background color should be constrained to the left pane only

## Current Behavior
- User message background spans the full terminal width
- The background color affects both the agent pane AND the kanban pane
- The kanban pane's background changes based on agent pane message positioning

## Visual Evidence
See attached screenshot showing the background color bleeding into the right pane where issue NOR-023 is visible.

## Technical Note
This is likely a rendering issue where the row background is being set instead of just styling the specific pane region. The background should be clipped or constrained to the agent pane's width boundaries.
