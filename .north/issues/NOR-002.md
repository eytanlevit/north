**Context**: Prevent accidental issue deletions in the TUI by requiring explicit confirmation before proceeding with the delete operation.

**Requirements**:
- Show centered overlay box confirmation dialog when delete is triggered in TUI
- Dialog clearly states which issue is being deleted (show ID and title)
- Provide "Confirm" and "Cancel" buttons
- Default action is Cancel (pressing Enter cancels, must explicitly confirm)
- Only proceed with deletion after explicit confirmation
- Support keyboard navigation (Tab to switch, Enter to select, Esc to cancel)
- CLI delete-issue command skips confirmation (no dialog in CLI mode)

**Acceptance Criteria**:
- Triggering delete in TUI shows centered overlay (not immediate deletion)
- User can navigate with keyboard (Tab, Enter, Esc)
- Default is Cancel (safe - prevents accidental Enter presses)
- Deletion only happens after explicit confirmation

**Technical Details**:
- Files: `src/tools/delete-issue.ts`, `src/components/kanban-pane.ts`
- Create confirmation overlay component (similar to issue detail overlay)
- Dialog layout:
  ```
  ┌──────────────────────┐
  │ Delete Issue?        │
  │ NOR-005: Rename...   │
  │ [Confirm] [Cancel]   │
  └──────────────────────┘
  ```
- Only show in TUI context, skip for CLI usage
- Integrate with existing overlay system
