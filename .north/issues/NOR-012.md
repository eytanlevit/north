Add ability to select an issue and launch a coding agent in the background using a git worktree and tmux session.

## Requirements
- Select an issue from the list
- Trigger "start in background" action (keyboard shortcut or command)
- Creates a git worktree for the issue
- Spawns a coding agent session in tmux
- Opens new tmux window/tab (if using cmux or similar)
- Uses the issue content as the initial prompt
- Agent works independently in the worktree

## Workflow
1. User selects issue NOR-005 in the list
2. Presses keybinding (e.g., `b` for background, or `s` for start)
3. System creates worktree: `git worktree add .worktrees/NOR-005`
4. System creates tmux session: `tmux new-session -d -s NOR-005`
5. System launches coding agent with issue as prompt
6. If in cmux, opens new tab showing the session
7. Issue status updated to "in-progress"

## Technical Considerations
- Git worktree management (naming, cleanup)
- Tmux session creation and attachment
- Which coding agent to use? (Claude Code, Cursor, custom?)
- How to pass issue content as initial prompt?
- Session lifecycle management
- Worktree cleanup when done
- Integration with cmux (if available)
