The kanban board's file watcher (`onIssueChange`) does not detect changes made to `.pm/issues/` files by external processes (e.g. the CLI, other editors). The board only updates when changes are made through the agent's own tool calls.

## Steps to Reproduce
1. Start the TUI (`npm run dev`)
2. In a separate terminal, update an issue via CLI: `npx tsx src/cli.ts update NOR-007 --status done`
3. Observe the kanban board — it does not reflect the change
4. Restart the TUI — the change is now visible

## Expected Behavior
The kanban board should auto-refresh when `.pm/issues/` files are modified by any process, not just agent tool calls.

## Investigation Notes
- `onIssueChange()` in `src/issues.ts` likely uses a callback pattern triggered only by `writeIssue()` calls within the same process
- Need to add an `fs.watch` / `chokidar` watcher on the `.pm/issues/` directory to detect external changes
- Should debounce to avoid excessive refreshes
