**Context**: Users should be able to attach files (images, documents, etc.) to issues for better context and documentation. Currently, issues only support markdown text in the body field.

**Requirements**:
- Add `attachments` field to issue schema (array of attachment objects)
- Create `.north/attachments/{ISSUE_ID}/` directory structure for storing files
- Support adding attachments via CLI: `north attach <ID> <filepath> [--description "text"]`
- Support viewing/listing attachments: `north show <ID>` should display attachment list
- In TUI, display attachments in issue detail view
- For images, support inline preview in TUI if possible
- Handle file copying/storage when attachments are added
- Support removing attachments: `north detach <ID> <attachment-name>`

**Attachment Schema**:
```yaml
attachments:
  - filename: screenshot.png
    originalPath: /path/to/original.png
    storedPath: .north/attachments/NOR-001/screenshot.png
    description: "Dashboard mockup"
    addedAt: 2026-03-03T12:00:00.000Z
    addedBy: user
```

**Acceptance Criteria**:
- [ ] Issue schema supports `attachments` field with proper validation
- [ ] CLI can add files to issues and copies them to `.north/attachments/{ID}/`
- [ ] CLI can list attachments when showing issue details
- [ ] CLI can remove attachments from issues
- [ ] TUI displays attached files in issue detail panel
- [ ] Image attachments show preview or can be opened
- [ ] Attachments are properly stored and referenced with relative paths
- [ ] `.gitignore` considerations documented (should attachments be committed?)
